# UI/UX-Konzept

## Grundprinzip

Die Oberfläche soll nicht wie ein komplexes Dokumentenmanagementsystem wirken, sondern wie ein ruhiger persönlicher Assistent.

Der Nutzer soll nach dem Öffnen sofort sehen:

- Was ist neu?
- Was ist dringend?
- Welche Fristen laufen?
- Welche Aufgaben sind offen?
- Welche Fälle brauchen Aufmerksamkeit?

## Hauptnavigation

Empfohlene Hauptbereiche:

1. Dashboard
2. Dokumente
3. Fälle
4. Personen
5. Organisationen
6. Fristen
7. Aufgaben
8. KI-Chat
9. Vorlagen / DokuForge
10. Einstellungen

## Dashboard

Das Dashboard zeigt:

- neue Dokumente
- kritische Fristen
- offene Aufgaben
- aktive Fälle
- zuletzt hinzugefügte Dokumente
- KI-Empfehlungen

Beispiel-KI-Karte:

> Kritisch: Beim Fall „Jobcenter 2026“ läuft am 15.05.2026 eine Frist ab. Empfohlen: Kontoauszüge hochladen und Antwort senden.

## Dokumentenansicht

Eine Dokumentenansicht sollte enthalten:

- Originaldokument als Vorschau
- OCR-Text
- KI-Zusammenfassung
- erkannte Felder
- Fristen
- Aufgaben
- zugeordnete Personen
- zugeordneter Fall
- Antwort generieren
- Frage zu diesem Dokument stellen

## Fallansicht

Eine Fallansicht sollte enthalten:

- Fallstatus
- Priorität
- Kurzbeschreibung
- nächste Aufgabe
- Fristen
- Chronologie
- Dokumente
- beteiligte Personen
- Chat mit diesem Fall
- Antwort generieren

## Personenansicht

Eine Personen- oder Organisationsansicht sollte enthalten:

- Stammdaten
- Dokumente
- Fälle
- offene Aufgaben
- Fristen
- Verlauf
- Notizen
- KI-Zusammenfassung

## KI-Chat

Der Chat sollte verschiedene Modi haben:

- Frage zu allen Dokumenten
- Frage zu einem Fall
- Frage zu einer Person
- Frage zu einem Dokument

Beispiele:

- „Was muss ich diese Woche erledigen?“
- „Fasse den Fall Jobcenter zusammen.“
- „Erstelle eine Antwort auf diesen Brief.“
- „Welche Fristen habe ich gegenüber der Krankenkasse?“

## Mobile-First-Erfassung

Da viele Dokumente fotografiert werden, sollte der Upload mobil besonders einfach sein:

1. Kamera öffnen
2. Dokument fotografieren
3. Bild automatisch zuschneiden
4. OCR starten
5. Analyse anzeigen
6. Nutzer bestätigt oder korrigiert Zuordnung
